## $A$-List

We manage the ùê¥-list as a stack, at the start of the program (A) we start pushing the variables, scope by scope.

When we want to look up a variable, we start popping all the way up the list until we find what we‚Äôre looking for (ex. **If I want y I need to check all the names before it while iterating through the stack**)

**Analysis for the** $A$**-List:**

- Simple to implement
- **Memory use:** Names are listed explicitly
- **Management costs:** Entering or exiting a scope causes the insertion or removal of stack elements
- **Access cost:** Linear in the depth of the ùê¥-list
- The average cost can be reduced but at the cost of increasing the cost of scope change
